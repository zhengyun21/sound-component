# 麦克风 + 时间轴功能设计文档

## 概述
在已有的声波可视化基础上，添加麦克风实时输入和时间轴显示功能，模拟完整的录音体验。

## 功能需求

### 1. 麦克风功能
- 使用 Web Audio API 获取麦克风输入
- 实时音量检测，驱动条形柱高度变化
- 音量平滑处理 + 背景噪音过滤
- 权限处理：拒绝时显示错误提示，保留滑块功能
- UI：圆形麦克风按钮，录音时变红色

### 2. 时间轴功能
- 显示录制总时长（MM:SS格式）
- 条形柱下方显示时间标记（MM:SS.ms格式）
- 时间标记随条形柱从左向右移动
- 时间轴与条形柱同步移动

## 技术实现

### 音频处理流程
```
用户点击麦克风按钮
    ↓
请求 getUserMedia 权限
    ↓
创建 AudioContext + AnalyserNode
    ↓
获取音频数据，计算 RMS 音量
    ↓
平滑处理 + 阈值过滤
    ↓
更新条形柱 + 时间标记
```

### 关键 API
- `navigator.mediaDevices.getUserMedia` - 获取麦克风
- `AudioContext` - 音频上下文
- `AnalyserNode` - 音频分析
- `requestAnimationFrame` - 60 FPS 更新

### 数据结构
```javascript
// 音量历史（含时间信息）
volumes = [
  { value: 0.3, time: '00:01.2' },
  { value: 0.5, time: '00:01.3' },
  // ...
]
```

### 视觉细节
- 条形柱：白色圆角矩形，4px宽，间距6px
- 时间标记：小号白色字体，居中对齐
- 麦克风按钮：圆形，录音时红色
- 背景：纯黑色

## 核心函数

### 1. `startMicrophone()`
- 请求麦克风权限
- 初始化 AudioContext
- 设置音频流分析

### 2. `stopMicrophone()`
- 停止音频流
- 清理资源

### 3. `calculateVolume()`
- 从 AnalyserNode 获取音频数据
- 计算 RMS 音量值
- 应用平滑和阈值

### 4. `updateTimeline()`
- 记录时间戳
- 更新时间标记
- 同步移动

## 错误处理

### 麦克风权限拒绝
- 显示友好的错误提示
- 自动切换回手动滑块模式
- 保留其他功能

### 音频处理失败
- 优雅降级
- 显示错误信息

## 性能考虑
- 使用 `requestAnimationFrame` 而非 `setInterval`
- 音频数据处理轻量高效
- DOM 操作最少化

## 浏览器兼容性

- Firefox: 支持
- Safari: 支持（需 HTTPS）
- 移动端：支持
